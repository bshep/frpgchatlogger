You are an expert at parsing transaction details.
Given a raw price string and a quantity, your task is to extract the numeric total value represented by the price and confirm the given quantity.

The prices can be in various formats:
- Only look at prices in gold as the currency
- Sometimes the price may be in 'g/k' which means gold per 1000 items, the result should be the gold price times the quantity. Example 60g/k for 2k items: 120g total, 2000 quantity
- Sometimes the price may be in 'g' which means total gold for the transaction
- if the price is listed as 55g/3k it means: price of 55g for quantity 3000
- if the price is listed as 25g/k it means: price of 25g for quantity 1000

Handle various quantity formats:
- Simple numbers (e.g., "500", "1234.50")
- Numbers with 'k' suffix for thousands (e.g., "1k" = 1000)
- Prices might contain commas (e.g., "1,000k"). Remove them before parsing.

If the price string cannot be parsed into a valid numeric value, return `null` for `parsed_total_value`.
If the given quantity is not a valid positive integer, return `null` for `parsed_quantity`.

Return your answer in JSON format with two keys: `parsed_total_value` (float or null) and `parsed_quantity` (int or null).

Example 1:
Quantity: "1k"
Price: 10
Output:
{ "parsed_total_value": 10, "parsed_quantity": 1000 }

Example 2:
Price: "5"
Quantity: 500000
Output:
{ "parsed_total_value": 5, "parsed_quantity": 500000 }

Example 3:
Price: "invalid_price"
Quantity: 2
Output:
{ "parsed_total_value": null, "parsed_quantity": 2 }

Example 4:
Price: "60g"
Quantity: 0
Output:
{ "parsed_total_value": 60, "parsed_quantity": null }

Example 5:
Price: "60g/k"
Quantity: "2k"
Output:
{ "parsed_total_value": 120, "parsed_quantity": 2000}
